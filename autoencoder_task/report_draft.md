# DNN実践課題2 第1題: AutoEncoderの学習

## 1. 目的

AutoEncoderを用いて、CIFAR-10画像を教師ラベルなしで復元する実験を行う。AutoEncoderは、入力画像をencoderで低次元のbottleneck特徴に変換し、decoderで元画像に戻すネットワークである。本実験では、復元画像の見た目と、bottleneck特徴を用いたk-meansクラスタリングの結果から、教師なしで得られた特徴量の性質を確認する。

## 2. 方法

データセットにはCIFAR-10を用いた。画像は3チャンネル32x32 pixelであり、入力画像そのものを教師信号として学習した。ネットワークは畳み込み層で画像を圧縮し、全結合層で64次元のbottleneck特徴に変換する。その後、全結合層と転置畳み込み層を用いて、入力と同じ3x32x32の画像を復元した。

損失関数には平均二乗誤差（MSE）を用い、optimizerにはAdamを用いた。学習後、test画像をencoderに入力して64次元特徴を取り出し、k-meansでk=5およびk=10のクラスタリングを行った。

## 3. 結果

学習中のMSEを確認すると、1 epoch目は0.03347017であり、10 epoch目には0.00823573まで低下した。したがって、epochが進むにつれてAutoEncoderが入力画像を復元するための特徴表現を学習できたと考えられる。

復元画像を以下に示す。

![reconstructions](figures/reconstructions.png)

復元画像では、入力画像の大まかな色や物体の位置は再現されている。一方で、細かい輪郭や背景の詳細はぼやけており、bottleneckで情報が圧縮されていることが確認できる。

k=5でクラスタリングした結果を以下に示す。

![kmeans k5](figures/kmeans_k5.png)

k=5の場合、ship, truck, airplane など、見た目や背景が比較的近いカテゴリが同じクラスタに集まる傾向が見られた。一方で、frog, cat, horse など異なるカテゴリも同じクラスタに含まれていた。

k=10でクラスタリングした結果を以下に示す。

![kmeans k10](figures/kmeans_k10.png)

k=10の場合、k=5よりもクラスタが細かく分かれたが、CIFAR-10の正解ラベルと完全に対応するわけではなかった。たとえば、truck, ship, airplane などが同じクラスタに含まれる一方で、cat, dog, frog, horse なども複数のクラスタに分散していた。

## 4. 考察

AutoEncoderは正解ラベルを使わずに、画像を復元するという目的だけで特徴表現を学習できる。復元に必要な情報はbottleneckに圧縮されるため、この特徴量には画像の色、形、構図などの情報が含まれると考えられる。

一方で、復元を目的とした特徴量は必ずしも分類に最適な特徴ではない。そのため、k-meansの結果では似たカテゴリの画像が近くなる場合もあるが、分類ラベルと完全に対応するとは限らない。より分類に有効な特徴を得るには、分類タスクで事前学習したCNN特徴や、contrastive learning、masked autoencoderなどの手法と比較する必要がある。

## 5. まとめ

CIFAR-10を用いて畳み込みAutoEncoderを学習し、入力画像の復元とbottleneck特徴のクラスタリングを行った。学習MSEは1 epoch目の0.03347017から10 epoch目の0.00823573まで低下した。復元画像から、AutoEncoderが画像の大まかな構造を学習できることが確認できた。また、k-meansの結果から、教師ラベルなしでも見た目が似た画像がある程度まとまることが分かった。
